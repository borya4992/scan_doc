# Деплой Scan Doc на Vercel — пошаговая инструкция

Приложение уже готово к деплою. Сборка проходит успешно.

---

## Способ 1 — самый простой (рекомендуется)

Git **не нужен**. Достаточно Node.js (у вас уже установлен).

### Шаг 1. Запустите деплой

Дважды щёлкните файл в папке проекта:

```
Deploy to Vercel.bat
```

Или в терминале:

```bash
cd "d:\проекты\Scan_doc"
npm run deploy
```

### Шаг 2. Войдите в Vercel

1. Откроется браузер
2. Если аккаунта нет — нажмите **Sign Up** (можно через Google или GitHub)
3. Подтвердите вход в терминале

### Шаг 3. Ответьте на вопросы CLI

При первом деплое Vercel спросит:

| Вопрос | Что выбрать |
|--------|-------------|
| Set up and deploy? | **Y** (Yes) |
| Which scope? | Ваш аккаунт |
| Link to existing project? | **N** (No) — первый раз |
| Project name? | `scan-doc` (или любое имя латиницей) |
| Directory? | **Enter** (текущая папка `.`) |
| Modify settings? | **N** (No) |

### Шаг 4. Получите ссылку

В конце появится строка вида:

```
Production: https://scan-doc-xxxxx.vercel.app
```

**Эту ссылку открывайте с телефона** — приложение работает 24/7, компьютер можно выключить.

### Повторный деплой (после изменений)

Снова запустите `Deploy to Vercel.bat` или:

```bash
npx vercel --prod
```

## Способ 2 — через сайт Vercel + GitHub

Если хотите управлять проектом через браузер.

### Шаг 1. Создайте репозиторий на GitHub

1. Зайдите на [github.com](https://github.com) → **Sign up** (если нет аккаунта)
2. **New repository** → имя `scan-doc` → **Create repository**
3. На странице репозитория: **uploading an existing file**
4. Перетащите **все файлы** из `d:\проекты\Scan_doc`, **кроме**:
   - папки `node_modules`
   - папки `.next`
   - файлов `.scan-doc.pid`, `.scan-doc.log`
5. **Commit changes**

### Шаг 2. Подключите Vercel

1. [vercel.com](https://vercel.com) → **Sign Up** (можно через GitHub)
2. **Add New…** → **Project**
3. **Import** репозиторий `scan-doc`
4. Настройки оставьте по умолчанию:
   - Framework: **Next.js**
   - Build Command: `next build`
   - Output: автоматически
5. **Deploy**

### Шаг 3. Готово

Через 1–2 минуты появится ссылка `https://scan-doc.vercel.app`.

При каждом обновлении файлов на GitHub Vercel пересоберёт сайт автоматически.

## Проверка после деплоя

1. Откройте ссылку на телефоне
2. Нажмите **Сфотографировать** или **Выбрать файлы**
3. Загрузите PDF или JPG
4. Нажмите **Экспорт в Excel**

## Частые проблемы

### «Need to log in»
Запустите в терминале:
```bash
npx vercel login
```

### Страница не открывается
Подождите 1–2 минуты после деплоя и обновите страницу.

### OCR медленно работает
Это нормально — распознавание идёт **на телефоне**, не на сервере Vercel. Для 500+ документов загружайте файлы пакетами.

### Бесплатный лимит Vercel
Для этого приложения лимита хватает: OCR на клиенте, сервер только отдаёт страницу.
