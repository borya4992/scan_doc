# Обновление GitHub после ошибки сборки на Vercel

Сборка упала, потому что на GitHub лежит **старая версия** проекта (с `exceljs`).
Локально проект уже исправлен и собирается без ошибок.

## Что сделать (5 минут)

### Вариант А — обновить файлы на GitHub через браузер

1. Откройте ваш репозиторий на GitHub
2. Обновите эти файлы (кнопка **Edit** или **Upload files**):

| Файл | Зачем |
|------|--------|
| `package.json` | без exceljs, с xlsx |
| `package-lock.json` | актуальные зависимости |
| `src/lib/excel.ts` | новый экспорт через xlsx |
| `next.config.ts` | настройки для Vercel |

3. **Commit changes** → Vercel пересоберёт сайт автоматически (1–2 мин)

### Вариант Б — загрузить проект заново

1. GitHub → ваш репозиторий → **Add file** → **Upload files**
2. Перетащите **всю папку** `d:\проекты\Scan_doc`, **кроме**:
   - `node_modules` (папка)
   - `.next` (папка)
   - `.scan-doc.pid`, `.scan-doc.log`
3. Если спросит про замену файлов — **Replace**
4. **Commit changes**

### Вариант В — деплой без GitHub (с компьютера)

Дважды щёлкните **`Deploy to Vercel.bat`** в папке проекта.

---

## Проверка

После обновления на Vercel:

- Статус: **Ready** (зелёный)
- Откройте ссылку `https://scan-doc-....vercel.app`

Если снова ошибка — откройте **Журналы развертывания**, прокрутите вниз до красных строк и пришлите скрин.
